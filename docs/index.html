üì• Installation

Requirements

  • Node.js 16+ installed
  • Java 17+ for running Minecraft servers
  • Windows, Linux, or macOS

Quick Start

# Clone the repository
git clone https://github.com/bySenom/MCRCON.git
cd MCRCON

# Install dependencies
npm install

# Copy environment template
cp .env.example .env

# Start the server
npm start

# Or use development mode with auto-reload
npm run dev

First Time Setup

1. Open your browser and navigate to http://localhost:3000

2. Register a new account (first user becomes admin)

3. Follow the interactive tutorial to create your first server

4. Configure your server settings and start managing!

Configuration

Edit .env file to customize:

  • PORT - Web interface port (default: 3000)
  • JWT_SECRET - Secret key for JWT authentication
  • SESSION_SECRET - Secret key for sessions

üìñ Usage Guide

Creating a Server

  1. Navigate to the "Create Server" tab
  2. Choose a template or create custom configuration
  3. Select server type (Vanilla, Paper, Fabric, etc.)
  4. Choose Minecraft version
  5. Configure host/IP, port, RAM allocation
  6. Set RCON credentials
  7. Click "Server erstellen" and wait for download to complete

Managing Servers

  • Start/Stop/Restart: Use control buttons on server cards or details page
  • Live Console: View real-time logs and execute commands
  • Settings: Edit server.properties through web interface
  • Files: Browse and edit server files directly
  • Backups: Create, restore, and download backups
  • Players: Manage players with 11 different actions
  • World: Download, reset worlds, edit gamerules and borders

Setting Up a Proxy Network

  1. Create a Velocity/Waterfall/BungeeCord proxy server
  2. Configure host/IP for external access
  3. Use "Create & Add" to create backend servers
  4. Backend servers automatically inherit proxy's IP
  5. First server becomes default, others require /server command

Installing Plugins/Mods

  1. Go to server details ‚Üí "Plugins/Mods" tab
  2. Search for plugins (auto-filtered by server type)
  3. Select compatible version
  4. Click install and restart server

Task Automation

  1. Navigate to "Tasks" tab
  2. Create new task (backup, restart, command)
  3. Set cron schedule (e.g., 0 */6 * * * = every 6 hours)
  4. Task runs automatically or trigger manually

Discord Notifications

  1. Go to "Webhooks" tab
  2. Add Discord webhook URL
  3. Select events to monitor
  4. Optionally link to specific server
  5. Test webhook and enable

üîå API Reference

REST API with JWT authentication. All endpoints require Authorization: Bearer <token> header.

Main Endpoints

  • GET /api/servers - List all servers
  • POST /api/servers/create - Create new server
  • POST /api/servers/:id/start - Start server
  • POST /api/servers/:id/stop - Stop server
  • POST /api/servers/:id/command - Execute command
  • GET /api/servers/:id/logs - Get server logs
  • GET /api/servers/:id/world/info - Get world information
  • POST /api/servers/:id/proxy/create-and-add - Create backend server
  • GET /api/plugins/search - Search plugins/mods
  • GET /api/tasks - List scheduled tasks
  • GET /api/webhooks - List webhooks
